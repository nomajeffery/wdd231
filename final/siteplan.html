MyShop Site Plan

Site Name

MyShop – Simple, memorable name that clearly communicates the purpose of an online shopping platform. Easy to remember and conveys accessibility for everyday shoppers.

Site Purpose

MyShop provides a modern, user-friendly e-commerce platform for customers to browse, search, filter, and purchase products. Features include:

  • Product browsing with category filtering (Electronics, Jewelery, Men's Clothing, Women's Clothing)
  • Search functionality to find products by title or keyword
  • Quick view modal for detailed product information
  • Shopping cart with add/remove functionality and quantity management
  • User authentication (login page)
  • Persistent cart storage using browser's LocalStorage
  • Responsive design for mobile, tablet, and desktop devices

Target Audience

  • Primary: General consumers aged 18-65 looking for convenient online shopping
  • Secondary: Tech-savvy users seeking quick product discovery and comparison
  • Tertiary: Mobile users shopping on-the-go with various devices

User Scenarios

  • Scenario 1: Sarah is a college student looking for a laptop. She searches for "laptop", filters by price, adds it to her cart, and purchases it.
  • Scenario 2: John wants to browse clothing. He filters by Men's Clothing, clicks "Quick View" on items, and adds favorites to cart for later checkout.
  • Scenario 3: Emma shops on mobile while commuting. She browses jewelry, maintains her cart across sessions with LocalStorage, and logs in to complete purchase.

Color Schema

Professional and modern color palette for e-commerce:

Primary Blue (#0056b3) – Buttons, links, and primary actions. Conveys trust and professional commerce.
Dark Blue (#003d82) – Hover states, secondary buttons, and emphasis. Clear feedback on interactions.
Alert Red (#c41e3a) – Error messages, remove buttons, and alerts. Draws attention to critical actions.
Dark Gray (#1a1a1a) – Primary text color for maximum readability and accessibility.

Key Features

  • Product API Integration: Fetches 20+ products from Fakestore API
  • Search & Filter: Real-time product search and category filtering
  • Quick View Modal: WCAG-compliant modal dialog for product details
  • Shopping Cart: Persistent cart using browser LocalStorage
  • Quantity Management: Adjust quantities and recalculate totals
  • Form Validation: Client-side validation with URLSearchParams
  • Responsive Design: Mobile-first with responsive breakpoints

Responsive Design Strategy

  • Mobile First: Designed for mobile, enhanced for larger screens
  • Breakpoints: Mobile (0-600px), Tablet (601-1024px), Desktop (1025px+)
  • Navigation: Hamburger menu on mobile, full horizontal menu on desktop
  • Layout Methods: CSS Flexbox and Grid for responsive layouts
  • No Horizontal Scrolling: All content fits within viewport

Accessibility Features (WCAG 2.1)

  • Semantic HTML5 elements (<header>, <main>, <footer>)
  • ARIA labels on all interactive elements
  • Skip link to main content
  • Proper heading hierarchy (H1, H2, H3)
  • Alt text on all images
  • Color contrast ratios meeting WCAG AA standards (4.5:1)
  • Focus states on all interactive elements
  • Form labels properly associated with inputs
  • Modal dialog with keyboard support (ESC to close)
  • Full keyboard navigation support

JavaScript Features

  • DOM Manipulation: Dynamic product card creation and content insertion
  • Array Methods: .filter() for search, .forEach() for iteration, .push()/.splice() for cart
  • Template Literals: Backticks and ${} for dynamic HTML generation
  • Async/Await: API calls with try/catch error handling
  • ES Modules: Modular code with export/import
  • Local Storage: Persistent cart data across sessions
  • Event Handling: Click, input, and keyboard events

Technical Implementation

  • HTML: Valid HTML5 with semantic structure and meta tags
  • CSS: External stylesheets with responsive media queries
  • JavaScript: Vanilla JavaScript for interactivity
  • API: Fakestore API for product data
  • Storage: Browser LocalStorage for cart persistence
  • Forms: URLSearchParams for form data transmission
  • Icons: Font Awesome 6.5.0 CDN

HTML & CSS Validation

  • Valid HTML5 markup with no syntax errors
  • Valid CSS with proper selectors and responsive queries
  • Semantic HTML elements for accessibility and SEO
  • Proper meta tags (charset, viewport, description, Open Graph, Twitter Card)
  • Mobile-responsive design tested on multiple devices
  • No console errors or JavaScript issues
  • Lighthouse audit scores of 95+ for Accessibility, Best Practices, and SEO
  • Zero color contrast issues at WCAG AA level
